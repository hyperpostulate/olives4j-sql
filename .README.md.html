Olives4J - SQL

A lightweight java library created to help building complex sql queries and manage binding as a more readable, writeable and maintainable way.

Overview

Sample code demonstrate using olives4j-sql api

    // create sql object for the query
    SQL sql = SQL.of("select * from customer where 1=1",  
                " and store_id   = :store_id",  
                " and active     = :active ",  
                " and create_date between :startdate and :enddate");

    // bind declared parameters
    SimpleDateFormat sdf=new SimpleDateFormat("yyyy-MM-dd");

    sql.bindings().bind("store_id" , null).optional();// mark store_id as optional
    sql.bindings().bind("active"   , true);
    sql.bindings().bind("startdate", sdf.parse("2006-02-14"));
    sql.bindings().bind("enddate"   , sdf.parse("2006-02-16"));

    // get the final query  
    String query = sql.toString();
    String queryFormatted = sql.format();
    System.out.println("SQL:\n"+query);
    System.out.println("SQL DEBUG:\n"+queryFormatted);

This code is produce the following output;

        SQL: 
        select * from customer where 1=1  and first_name like (?)  and active    = (?)   and create_date between (?)  and (?)

        SQL DEBUG: 
        select * 
        from customer 
        where 1=1 
        and active = (? /* true*/) 
        and create_date between (? /* Sat Nov 12 22:20:02 EET 2016*/) 
        and (? /* Sat Nov 12 22:20:02 EET 2016*/)

As you may realize the following line removed from final query. 

    and store_id   = :store_id

Observations from the sample code;

  • We use SQL type to build an sql statement instead of StringBuilder/StringBuffer
  • SQL type accepts varargs parameters for its builder methods
  • SQL type support named binding variables
  • Binding variables may be defined as optional
  • SQL expressions including bindings defined as optional removed from the final query if their value set to null
  • SQL format() produce a reformatted sql string including binding parameter values, makes debugging a lot easier

Olives4j-sql library contains

  • SQL Script parser
  • STree data structure to build dynamic structural string
  • SQL type based on Stree data structure
  • Some utilities which handle with sql strings

Usage

There are three technique you can build dynamic sql queries with olives4j-sql

  1. provide binding parameters while building sql
        import static tr.com.olives4j.sql.SQL.$;
        SQL sql = SQL.of(
                "select * from customer where 1=1", 
                " and store_id   ="         ,$(null).optional(true), 
                " and first_name like "     ,$("a%"), 
                " and active     = "        ,$(true), 
                " and create_date between " ,$(new Date()), " and ",$(new Date()));
  1. Build sql with named parameters and bind later
    SQL sql = SQL.of("select * from customer where 1=1",  
                " and store_id   = :store_id",  
                " and active     = :active ",  
                " and create_date between :startdate and :enddate");

    sql.bindings().bind("store_id" , null).optional(true);
    sql.bindings().bind("active"   , true);
    sql.bindings().bind("startdate", new Date());
    sql.bindings().bind("enddate"  , new Date());
  1. Use an external sql script file with named parameters

-- app.sql file content -------------------

    --@NAMED search_customer
    select * 
    from customer 
    where 1=1 
    and store_id=:store_id
    and first_name like :first_name
    and active = :active 
    and create_date between :create_date_from and :create_date_to
    ;

    --@NAMED find_customer_by_id
    select * 
    from customer 
    where 1=1 
    and customer_id=:customer_id
    ;
    ....

-- Java code ------------------------------ 

    // Parse sql script file 
    SQLCollection sqls = SQLReader.read(new FileReader("src/test/sql/test1-search-query.sql"));

    SQL sql = sqls.get("search_customer");
    sql.bindings().bind("store_id", null).optional(true);
    sql.bindings().bind("active"   , true);
    sql.bindings().bind("startdate", new Date());
    sql.bindings().bind("enddate"  , new Date());

SQL Annotation

Olives4j-sql is intended to be able to parse any ordinary sql/plsql script file. Besides that, it use an extension concept to add extra behaviour to sql queries while keeping the query as is The concept is like javadoc annotations

  • SQL annotations defined in line or block comments

  • SQL comments must be started with @ sign to be annotation

    --@ /*@

SQL api provides #getAnnotations() method to inquire the annotations. There are a few built in annotations used by the library.

  • @NAMED used to name sql queries.SQLReader assign the name to parsed SQL types if the @NAMED annotation defined.
